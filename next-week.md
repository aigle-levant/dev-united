If I had another week to continue working on this project, these would be my priorities.

## 1. Improve Entity Resolution

The current implementation uses deterministic confidence scoring based on signals such as usernames, websites, locations, and name similarity. While this is explainable and fast, it struggles with ambiguous cases.

I would improve this by:

- Using embeddings to compare profile descriptions and activity.
- Introducing Gemini as a tie-breaker only when confidence falls within an ambiguity range.
- Learning confidence weights from real-world data instead of manually assigning them.

## 2. Background Processing

Currently, profile ingestion happens synchronously during the API request.

I would move this into a background job system (Celery, Redis Queue, or FastAPI BackgroundTasks) so that:

- API responses return immediately.
- Long-running API calls do not block requests.
- Failed jobs can be retried independently.

## 3. Better Rate-Limit Handling

While GitHub rate-limit information is already exposed through the health endpoint, external APIs can still fail due to request quotas.

Improvements would include:

- Automatic retries with exponential backoff.
- Local caching for frequently requested profiles.
- Graceful degradation when one provider is unavailable.
- Provider-specific circuit breakers.

## 4. Expand Observability

The current `/health` endpoint provides API usage, GitHub rate limits, LLM usage, and resolution performance.

I would expand this with:

- Error rates by provider.
- Average latency per external API.
- Cache hit/miss statistics.
- Request success percentages.
- Historical metrics dashboards.

## 5. Additional Data Sources

The architecture was designed so additional providers can be integrated with minimal changes.

Potential additions include:

- LinkedIn (where permitted)
- Reddit
- Kaggle
- GitLab
- Medium
- Hashnode

Adding another provider would primarily require implementing an integration module and a normalization layer.

## 6. Testing

The current project contains basic validation, but I would significantly improve automated testing.

Areas to cover include:

- Integration tests
- Entity resolution edge cases
- API endpoint testing
- Database operations
- Mocking external APIs
- Load testing

## 7. Better AI Enrichment

Currently, Gemini generates a concise summary of the unified profile.

Future improvements include:

- Skill extraction
- Experience timeline generation
- Technology categorization
- Open-source contribution summaries
- Confidence explanations generated by the LLM

## 8. Security & Performance

Additional improvements would include:

- Authentication for API endpoints.
- Request rate limiting.
- Response caching.
- Asynchronous HTTP requests.
- Database indexing optimization.
- Bulk profile ingestion.
